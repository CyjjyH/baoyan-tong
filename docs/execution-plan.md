# 开发执行计划 — 保研信息聚合平台

> 版本：v1.0 | 创建日期：2026-06-30

---

## 总体原则

1. **小步快跑**：每个阶段产出可用的东西，不追求一步到位
2. **先骨架后血肉**：先跑通主流程，再加细节功能
3. **每阶段验收**：当前阶段完成并验证后，再进入下一阶段
4. **安全内建**：不等到最后才考虑安全

---

## 阶段 0：项目初始化（预计 1 天）

### 目标
把项目跑起来，能本地看到页面。

### 任务清单
- [ ] `npx create-next-app` 创建项目
- [ ] 配置 Tailwind CSS
- [ ] 安装 shadcn/ui，初始化
- [ ] 配置 Prisma + PostgreSQL（先用本地或云数据库免费层）
- [ ] 设计数据库表结构（通知表、用户表、收藏表、订阅表）
- [ ] 配置 NextAuth.js 基础框架
- [ ] 搭建项目目录结构
- [ ] 创建首页空白框架（Header + 布局）
- [ ] 配置 ESLint + Prettier
- [ ] 初始化 Git 仓库，首次提交

### 验收标准
- `npm run dev` 能正常启动
- 浏览器打开能看到基本页面布局
- 数据库连接正常

## 阶段 1：数据层 + 管理后台（预计 2-3 天）

### 目标
管理员能在后台录入通知，数据写入数据库并能在前端展示。

### 任务清单
- [ ] 实现通知表 CRUD API（`/api/notices`）
- [ ] 搭建管理后台页面（简单的表格 + 表单）
- [ ] 实现通知新增/编辑表单（所有字段）
- [ ] 首页展示通知列表（纯展示，暂无筛选）
- [ ] 通知详情页（展示所有字段）
- [ ] 管理后台添加基础权限（简单的 admin key 验证）

### 验收标准
- 后台能成功录入通知
- 首页能看到通知列表
- 点击通知能进入详情页
- 数据刷新后不丢失

---

## 阶段 2：搜索筛选 + 页面完善（预计 2-3 天）

### 目标
首页具备完整的搜索筛选能力，页面视觉完善。

### 任务清单
- [ ] 实现关键词搜索（数据库模糊查询）
- [ ] 实现学校等级筛选
- [ ] 实现省份/城市筛选
- [ ] 实现通知类型筛选（夏令营/预推免）
- [ ] 实现报名时间范围筛选
- [ ] 实现学校名称搜索
- [ ] 筛选条件联动（选择省份后城市自动缩窄）
- [ ] 完善首页 UI（按设计规范）
- [ ] 完善详情页 UI
- [ ] 移动端适配（筛选条件折叠）
- [ ] 响应式布局验证

### 验收标准
- 所有筛选条件可用，组合筛选结果正确
- PC 和手机端都能正常使用
- 页面视觉符合蓝白风格设计规范

## 阶段 3：用户系统（预计 2-3 天）

### 目标
用户可以注册登录、收藏通知、订阅提醒。

### 任务清单
- [ ] 完善用户表 + NextAuth 邮箱注册/登录
- [ ] 注册页 + 登录页 UI
- [ ] 邮箱验证流程
- [ ] 收藏功能（收藏按钮 + 收藏列表页）
- [ ] 订阅功能（设置条件 + 保存）
- [ ] 个人中心页面（我的收藏、我的订阅）
- [ ] 邮件提醒：有新匹配通知时自动发送

### 验收标准
- 可以成功注册并登录
- 可以收藏和取消收藏
- 可以创建订阅，收到匹配的邮件提醒

---

## 阶段 4：AI 自动搜集（预计 3-4 天）

### 目标
AI 每天自动搜索高校通知，提取结构化信息并入库。

### 任务清单
- [ ] 设计 AI 搜索 prompt（搜索词 + 提取字段要求）
- [ ] 实现 AI 搜索 + 信息提取 pipeline
- [ ] 实现去重判断逻辑
- [ ] 结果写入待审核库
- [ ] 管理后台添加"AI 结果抽查"页面
- [ ] 配置定时任务（每天一次）
- [ ] 手动触发搜索的按钮（调试用）
- [ ] 异常处理和重试机制
- [ ] 搜索日志记录

### 验收标准
- AI 能搜索到真实的高校夏令营通知
- 提取的信息准确率 > 80%
- 管理员能在后台快速审核 AI 结果
